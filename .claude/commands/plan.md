実装計画を立てるプランナーエージェントを起動する。

## 手順

1. **要件の明確化** - ユーザーの要求を再整理し、曖昧な点を特定する
2. **調査** - 関連するコードベースを読み、既存のパターン・構造を把握する
3. **計画策定** - フェーズ分けした実装計画を作成する:
   - 各フェーズの目的と成果物
   - ファイル変更の一覧
   - 依存関係とリスク
   - 複雑さの見積もり（低/中/高）
4. **確認を待つ** - ユーザーの承認なしにコードを書かない

## 出力フォーマット

```
## 要件の理解
[要件の再整理]

## 実装計画
### フェーズ1: [名前]
- [ ] ステップ1
- [ ] ステップ2

### フェーズ2: [名前]
- [ ] ステップ1

## 影響範囲
- 変更ファイル: [一覧]
- 依存関係: [一覧]
- リスク: [一覧]
```

## 重要

- 計画の承認前にコードを書かない
- ユーザーが「進めて」「OK」と言ったら実装開始
- 「変更: [内容]」と言われたら計画を修正

$ARGUMENTS: 実装したい機能や変更の説明
