現在のセッションから再利用可能なパターンを抽出し、スキルとして保存する。

## 抽出対象

1. **エラー解決パターン**
   - 発生したエラーとその根本原因
   - 解決方法
   - 同様のエラーに再利用できるか

2. **デバッグテクニック**
   - 非自明なデバッグ手順
   - 有効だったツールの組み合わせ

3. **ワークアラウンド**
   - ライブラリの癖
   - APIの制限事項
   - バージョン固有の修正

4. **プロジェクト固有のパターン**
   - コードベースの規約
   - アーキテクチャの決定事項
   - 統合パターン

## 手順

1. セッションを振り返り、抽出可能なパターンを特定
2. 最も価値のある知見を選定
3. スキルファイルを作成
4. ユーザーに確認を求める
5. `skills/learned/` に保存

## 出力フォーマット

```yaml
---
name: "[パターン名]"
description: "[説明]"
model: sonnet-4-5
---
```

```markdown
# [パターン名]

**抽出日:** [日付]
**コンテキスト:** [適用場面]

## 問題
[解決する問題]

## 解決策
[パターン/テクニック]

## 使用例
[コード例]

## 適用条件
[このスキルを使うべき状況]
```

## 注意

- 些細な修正（タイポ、構文エラー）は抽出しない
- 一時的な問題（API障害等）は抽出しない
- 1つのスキルに1つのパターンに絞る
