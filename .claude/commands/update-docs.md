ソースコードから最新のドキュメントを生成・更新する。

## 手順

1. **package.jsonの解析**
   - scriptsセクションからコマンド一覧を生成
   - 依存関係の一覧を整理

2. **環境変数の整理**
   - `.env.example` から全環境変数を抽出
   - 目的とフォーマットをドキュメント化

3. **READMEの更新**
   - セットアップ手順が最新か確認
   - 利用可能なスクリプト一覧を更新
   - 環境変数の説明を更新

4. **CLAUDE.mdの更新**
   - リポジトリ構造を現在の状態に更新
   - 新しいパターンや規約を反映
   - 技術スタック情報を更新

5. **陳腐化したドキュメントの検出**
   - 90日以上更新されていないドキュメントを特定
   - 手動レビューのためのリストを作成

6. **変更差分の表示**

## 重要

- 情報源はソースコード（package.json、.env.example、ディレクトリ構造）
- 推測でドキュメントを書かない（コードを読んで確認する）
- 既存のドキュメントのスタイルと一貫性を保つ
