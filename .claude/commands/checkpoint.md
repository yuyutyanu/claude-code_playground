作業のチェックポイントを作成・検証する。

## 使い方

$ARGUMENTS: `create <名前>` / `verify <名前>` / `list`

## create - チェックポイント作成

1. `git status` で現在の状態がクリーンか確認
2. 現在の状態でgitコミットまたはstashを作成
3. チェックポイントをログに記録:
   ```
   [日時] | [名前] | [git SHA]
   ```
4. 作成完了を報告

## verify - チェックポイントとの比較

1. 指定されたチェックポイントのSHAを取得
2. 現在の状態と比較:
   - 追加されたファイル
   - 変更されたファイル
   - テスト通過率の変化
   - ビルド状態

出力:
```
## チェックポイント比較: [名前]
- ファイル変更: N件
- テスト: +X通過 / -Y失敗
- ビルド: PASS/FAIL
```

## list - チェックポイント一覧

保存済みチェックポイントを表示:
- 名前
- タイムスタンプ
- Git SHA

## 典型的なワークフロー

```
/checkpoint create "機能実装開始"
  ↓ [実装]
/checkpoint create "コア実装完了"
  ↓ [テスト]
/checkpoint verify "コア実装完了"
  ↓ [リファクタリング]
/checkpoint create "リファクタリング完了"
  ↓ [PR作成]
/checkpoint verify "機能実装開始"
```
