コードベースのセキュリティスキャンを実行する。

## スキャン項目

### 1. 依存関係の脆弱性
- `npm audit` / `pnpm audit` で既知の脆弱性をチェック
- 重要度（critical/high/moderate/low）で分類

### 2. シークレットの漏洩
- ハードコードされたAPIキー、トークン、パスワード
- `.env` ファイルがgit管理されていないか確認
- `.gitignore` の設定確認

### 3. OWASP Top 10
- **インジェクション**: SQL、NoSQL、OS、LDAPインジェクション
- **認証の不備**: セッション管理、パスワードポリシー
- **機密データの露出**: 暗号化、転送中の保護
- **XXE**: XML外部エンティティ
- **アクセス制御の不備**: 認可チェック
- **セキュリティの設定ミス**: デフォルト設定、不要な機能
- **XSS**: DOM、Reflected、Stored
- **安全でないデシリアライゼーション**
- **既知の脆弱性を持つコンポーネント**
- **不十分なログと監視**

### 4. コードパターンの検査
- `eval()`, `innerHTML`, `dangerouslySetInnerHTML` の使用
- ユーザー入力の未サニタイズ使用
- CORS設定の確認
- CSRFトークンの確認

## 出力フォーマット

```
## セキュリティスキャンレポート

### CRITICAL (即座に修正が必要)
- [問題] [ファイル:行] [修正案]

### HIGH (マージ前に修正)
- [問題] [ファイル:行] [修正案]

### MEDIUM (計画的に修正)
- [問題] [ファイル:行] [修正案]

### LOW (推奨事項)
- [問題] [ファイル:行] [修正案]

### 依存関係
- critical: N件, high: N件, moderate: N件

## 総合判定: PASS / FAIL
```
