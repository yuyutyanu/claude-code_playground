コードベースの品質検証を実行する。

## 検証ステップ（順番に実行）

1. **ビルド検証** - `npm run build` または該当するビルドコマンド
2. **型チェック** - `npx tsc --noEmit` またはプロジェクトの型チェックコマンド
3. **リンター** - `npm run lint` またはプロジェクトのlintコマンド
4. **テスト実行** - `npm run test` でテストスイートを実行
5. **console文の監査** - console.log/warn/errorの残存チェック
6. **git状態の確認** - 未コミットの変更、未追跡のファイル

## 実行モード

$ARGUMENTS で以下のモードを指定:

- `quick` - ビルドと型チェックのみ
- `full` - 全項目を実行（デフォルト）
- `pre-commit` - コミット前チェック（ビルド、型、リント、テスト）
- `pre-pr` - PR前チェック（全項目 + セキュリティスキャン）

## 出力フォーマット

```
## 検証レポート

| 項目 | 結果 |
|------|------|
| ビルド | ✅ PASS / ❌ FAIL |
| 型チェック | ✅ PASS / ❌ FAIL |
| リンター | ✅ PASS / ❌ FAIL (警告N件) |
| テスト | ✅ PASS / ❌ FAIL (N/M通過) |
| console文 | ✅ なし / ⚠️ N件検出 |
| git状態 | ✅ クリーン / ⚠️ 未コミット変更あり |

## 判定: PR準備完了 / 要修正
```

## 重要

- ビルドが失敗したら以降のチェックをスキップしてエラー報告
- 各チェックの詳細なエラー内容を報告する
