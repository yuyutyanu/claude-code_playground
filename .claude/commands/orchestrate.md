複数のエージェントを順番に実行して、複雑なタスクを処理する。

## 使い方

$ARGUMENTS: `[ワークフロータイプ] [タスクの説明]`

## 定義済みワークフロー

### feature - 機能実装
```
/plan → /tdd → /code-review → /verify
```
新機能の計画から実装、レビュー、検証までの一連のフロー。

### bugfix - バグ修正
```
/plan → /tdd → /build-fix → /verify
```
バグの調査、テストによる再現、修正、検証のフロー。

### refactor - リファクタリング
```
/verify → /refactor → /verify → /code-review
```
現状確認→安全なリファクタリング→再検証→レビュー。

### release - リリース準備
```
/verify full → /test-coverage → /code-review → /update-docs
```
全検証、カバレッジ確認、レビュー、ドキュメント更新。

## 実行ルール

1. 各ステップの結果を次のステップに引き継ぐ
2. いずれかのステップが失敗したら停止して報告
3. 各ステップの結果サマリーを記録

## 最終レポート

```
## オーケストレーションレポート

### サマリー
[1段落の概要]

### 各ステップの結果
1. [ステップ名]: ✅/❌ [詳細]
2. [ステップ名]: ✅/❌ [詳細]

### 変更ファイル
- [ファイル一覧]

### 判定: SHIP / NEEDS WORK / BLOCKED
```
