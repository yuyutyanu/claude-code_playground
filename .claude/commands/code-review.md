コミットされていない変更に対してコードレビューを実行する。

## レビュー項目

### 1. セキュリティ（CRITICAL）
- ハードコードされた認証情報・APIキー・トークン
- SQLインジェクション、XSS、コマンドインジェクション
- 入力バリデーションの欠如
- パストラバーサル
- 安全でない依存関係

### 2. コード品質（HIGH）
- 50行を超える関数
- 800行を超えるファイル
- ネスト深度4以上
- エラーハンドリングの欠如
- デバッグ用コード（console.log等）の残存
- TODO/FIXME/HACKコメント

### 3. ベストプラクティス（MEDIUM）
- イミュータビリティパターンの遵守
- テストカバレッジの不足
- 型安全性（any型の使用等）
- アクセシビリティ（a11y）

## 手順

1. `git diff` と `git diff --staged` で変更内容を取得
2. 各変更ファイルを読み、変更のコンテキストを理解
3. 上記の項目に基づいてレビュー
4. レポートを出力

## 出力フォーマット

```
## コードレビュー結果

### CRITICAL
- [ファイル:行番号] [問題の説明] → [修正案]

### HIGH
- [ファイル:行番号] [問題の説明] → [修正案]

### MEDIUM
- [ファイル:行番号] [問題の説明] → [修正案]

## 判定: ✅ 承認 / ❌ 要修正
```

## 重要

- CRITICALまたはHIGHがある場合はコミットをブロック
- セキュリティ脆弱性のあるコードは絶対に承認しない
